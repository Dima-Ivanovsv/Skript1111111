Тип конфигурации: Клиентская конфигурация OpenVPN

Итоговый отчет:
В конфигурации присутствуют небезопасные параметры. Рекомендуется скорректировать настройки.
Параметров в состоянии Хорошо: 15, Удовлетворительно: 4, Плохо: 1, Критично для безопасности: 0.

ДЕТАЛЬНЫЙ АНАЛИЗ ПАРАМЕТРОВ:

[Хорошо] proto
    proto udp — используется UDP, рекомендованный для OpenVPN протокол.

[Хорошо] dev
    dev tun — используется туннельный режим (L3), обычно предпочтительнее.

[Хорошо] remote
    remote задан — клиент знает адрес сервера. Конкретные значения требуют ручной проверки.

[Хорошо] topology
    topology subnet — современный и рекомендуемый режим адресации.

[Хорошо] data-ciphers
    В data-ciphers присутствуют современные шифры (AES-GCM/CHACHA20-POLY1305). Это соответствует актуальным рекомендациям.

[Хорошо] auth
    auth SHA256 — современный и безопасный алгоритм HMAC.

[Хорошо] replay-window
    Параметр replay-window задан — включена защита от повтора пакетов.

[Хорошо] mute-replay-warnings
    mute-replay-warnings не используется — предупреждения о повторах будут отображаться в логах.

[Хорошо] PKI/static key
    Используется инфраструктура открытых ключей (PKI): заданы ca, cert, key.

[Хорошо] tls-version-min
    tls-version-min 1.2 — минимальная версия TLS 1.2, это приемлемый уровень.

[Хорошо] tls-auth/tls-crypt
    Используется tls-crypt/tls-crypt-v2 — управляющий канал шифруется и аутентифицируется.

[Удовлетворительно] dh/ecdh-curve
    Параметры DH/ECDH не заданы явно. Необходимо убедиться, что используется безопасная группа по умолчанию.

[Хорошо] remote-cert-tls server
    Клиент проверяет, что подключается именно к сертификату сервера (remote-cert-tls server).

[Удовлетворительно] keepalive/ping/ping-restart
    Не заданы keepalive/ping/ping-restart для клиента — допустимо, но можно улучшить устойчивость.

[Хорошо] persist-key/persist-tun
    persist-key и persist-tun включены — интерфейс и ключи не пересоздаются при перезапуске процесса.

[Удовлетворительно] log/log-append/status
    На клиенте не настроено отдельное логирование — это допустимо, но усложняет диагностику.

[Удовлетворительно] verb
    verb не задан — используется уровень логирования по умолчанию.

[Плохо] auth-nocache
    auth-user-pass используется без auth-nocache — пароль может кэшироваться в памяти клиента.

[Хорошо] nobind
    nobind включён — клиент не закрепляет локальный порт, что удобно за NAT/файрволами.

[Хорошо] verify-x509-name
    verify-x509-name задан — клиент дополнительно проверяет идентичность сертификата сервера.
